## Telegram English Bot

Дружелюбный Telegram-бот для практики английского языка на Python с использованием **Timeweb AI-агента** (модель **Qwen 3.5 Flash**).

Пользователь пишет сообщения на английском, бот:
- **отвечает на английском**;
- **поддерживает свободный разговор**;
- **задаёт встречный вопрос**;
- **исправляет ошибки** и **кратко объясняет их по-русски**.

Формат ответа бота:

Ответ:  
\<естественный ответ на английском, с встречным вопросом\>

Исправления:  
❌ ошибка пользователя  
✅ правильный вариант  

Пояснение:  
\<короткое объяснение на русском\>

---

### 1. Требования

- Python **3.11+**
- Аккаунт в [Timeweb Cloud](https://timeweb.cloud/) и созданный **AI-агент** с моделью **Qwen 3.5 Flash**
- **API-токен** и **ID агента** из панели Timeweb (см. ниже)
- **Telegram Bot Token**, полученный через [@BotFather](https://t.me/BotFather)

---

### 2. Подключение Timeweb AI-агента

Бот использует [Timeweb AI-агенты](https://timeweb.cloud/docs/ai-agents) и [OpenAI-совместимый API](https://timeweb.cloud/docs/ai-agents/api-usage/openai-compatible-api). Полная документация API: [agent.timeweb.cloud/docs](https://agent.timeweb.cloud/docs).

#### Что нужно сделать

1. **Войти в Timeweb Cloud**  
   Зарегистрируйтесь или войдите на [timeweb.cloud](https://timeweb.cloud/).

2. **Создать AI-агента**  
   - В разделе **«AI-агенты»** нажмите **«Создать агента»**.  
   - Выберите модель **Qwen 3.5 Flash** (или другую подходящую модель).  
   - Обязательно настройте **системный промпт** (инструкции агента) — см. раздел **«Настройка системного промпта в Timeweb»** ниже.

3. **Узнать ID агента**  
   - Откройте карточку агента.  
   - **ID агента** отображается в дашборде (вкладка «Дашборд» / «Обзор») или в URL страницы агента.  
   - Этот идентификатор понадобится для переменной `TIMEWEB_AGENT_ID` в `.env`.

4. **Создать API-ключ (токен)**  
   - В управлении агентом откройте раздел **«Ключи доступа»** ([документация](https://timeweb.cloud/docs/ai-agents/manage-agents/api-access-key)).  
   - Создайте новый ключ и **скопируйте токен**.  
   - Сохраните его в переменной `TIMEWEB_API_TOKEN` в `.env` (никому не передавайте и не коммитьте в репозиторий).

5. **Проверить базовый URL**  
   Для OpenAI-совместимого API используется фиксированный хост:  
   `https://agent.timeweb.cloud`  
   В коде бота URL формируется автоматически по `TIMEWEB_AGENT_ID`.

---

### 3. Настройка системного промпта в Timeweb

Системный промпт (роль, задачи и формат ответа) настраивается **на стороне сервиса Timeweb**: панель агента → **«Инструкции»** / **«Системный промпт»** ([документация](https://timeweb.cloud/docs/ai-agents/manage-agents/instructions)). Бот не подставляет свой промпт в запрос — используется только то, что задано в панели.

#### Что указать в промпте

1. **Роль** — кто агент (дружелюбный преподаватель английского для русскоязычных в Telegram).
2. **Задачи** — отвечать на английском, вести диалог, задавать встречный вопрос, исправлять ошибки и кратко объяснять по-русски.
3. **Формат ответа** — строго по блокам: «Ответ:» (английский), «Исправления:» (❌/✅), «Пояснение:» (русский).
4. **Ограничение длины** — явно попросить модель отвечать кратко (цель 250–300 токенов на ответ), чтобы не сжигать пакет токенов слишком быстро.

Готовый текст для копирования в панель Timeweb лежит в файле **`timeweb_system_prompt.txt`** в корне проекта. Откройте его, скопируйте содержимое и вставьте в поле системного промпта (инструкций) вашего агента в Timeweb Cloud.

#### Ограничение токенов в коде

В **`ai_service.py`** задана константа **`MAX_COMPLETION_TOKENS = 320`** — максимальное число токенов в одном ответе агента. Так вы ограничиваете расход токенов пакета. При необходимости значение можно изменить (рекомендуемый диапазон 250–350 для коротких ответов с исправлениями).

### 4. Структура проекта

В каталоге проекта (например, `English-Bot/` или `telegram_english_bot/`):

- `bot.py` — основной файл Telegram-бота (точка входа)
- `ai_service.py` — обращение к Timeweb AI через OpenAI-совместимый API (Chat Completions)
- `config.py` — загрузка переменных окружения из `.env`
- `prompts.py` — текст промпта для справки (основная настройка — в панели Timeweb)
- `timeweb_system_prompt.txt` — готовый системный промпт для вставки в панель Timeweb
- `database.py` — хранение диалогов и статистики (SQLite)
- `requirements.txt` — зависимости Python
- `.env.example` — пример файла переменных окружения

В корне:

- `README.md` — эта инструкция

### 5. Настройка окружения

Перейдите в каталог проекта:

```bash
cd "English-Bot"
```

Создайте виртуальное окружение (Windows, PowerShell):

```bash
python -m venv .venv
.venv\Scripts\activate
```

Установите зависимости:

```bash
pip install -r requirements.txt
```

---

### 6. Настройка `.env`

1. В каталоге проекта создайте файл `.env` на основе `.env.example`:

```bash
copy .env.example .env
```

2. Откройте `.env` и подставьте свои значения:

```env
# Timeweb AI-агент (ID агента и API-токен из панели Timeweb)
TIMEWEB_AGENT_ID=ваш-id-агента-из-дашборда
TIMEWEB_API_TOKEN=ваш-api-токен-из-раздела-ключи-доступа

# Telegram
TELEGRAM_BOT_TOKEN=ваш-токен-от-BotFather
```

- **TIMEWEB_AGENT_ID** — ID созданного AI-агента (Qwen 3.5 Flash).  
- **TIMEWEB_API_TOKEN** — токен из раздела «Ключи доступа» агента.  
- **TELEGRAM_BOT_TOKEN** — токен бота из [@BotFather](https://t.me/BotFather).

Не добавляйте `.env` в git (он уже в `.gitignore`).

### 7. Запуск бота

Из каталога проекта с активированным виртуальным окружением:

```bash
python -m bot
```

или:

```bash
python bot.py
```

После запуска:
- найдите бота в Telegram по имени;
- напишите сообщение **на английском**;
- бот ответит на английском, задаст встречный вопрос и при необходимости покажет блок с исправлениями и пояснением на русском.

### 8. Как это работает

- Бот использует **aiogram** для работы с Telegram API.
- **python-dotenv** подгружает переменные из `.env`.
- В **ai_service.py** бот отправляет запросы к Timeweb AI по **OpenAI-совместимому API**:
  - URL: `https://agent.timeweb.cloud/api/v1/cloud-ai/agents/{agent_id}/v1/chat/completions`
  - Аутентификация: заголовок `Authorization: Bearer <TIMEWEB_API_TOKEN>`
  - В теле запроса передаётся только **история диалога** (`messages`: user/assistant). Системный промпт задаётся в панели Timeweb и подставляется сервисом.
  - Параметр **`max_tokens`** ограничен константой `MAX_COMPLETION_TOKENS` (320), чтобы экономить пакет токенов.
- Модель (Qwen 3.5 Flash) и системный промпт (роль, задачи, формат ответа) настраиваются в панели агента Timeweb.

Если бот не запускается:
- убедитесь, что активировано виртуальное окружение и установлены зависимости;
- проверьте, что в `.env` верно указаны `TIMEWEB_AGENT_ID`, `TIMEWEB_API_TOKEN` и `TELEGRAM_BOT_TOKEN`.

Если бот молчит или выдаёт ошибку при ответе:
- проверьте, что агент в панели Timeweb активен и по нему не исчерпан лимит;
- при необходимости подождите и повторите запрос — API может быть временно недоступен.
